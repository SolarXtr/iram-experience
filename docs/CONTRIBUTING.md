# Contributing Guide

Thank you for your interest in contributing to the iRAM Experience project.

This document describes the recommended workflow for contributing code, documentation, and improvements.

---

# Development Workflow

```
Feature Branch

↓

Development

↓

Local Testing

↓

Commit

↓

Push

↓

Pull Request / Review

↓

Merge into master
```

---

# Branch Naming

Feature

```
feature/<feature-name>
```

Example

```
feature/frontend
feature/dashboard
feature/rmis
```

Bug Fix

```
fix/<issue-name>
```

Documentation

```
docs/<topic>
```

Release

```
release/<version>
```

---

# Commit Convention

Examples

```
feat: add landing page

fix: resolve navigation bug

docs: update architecture

style: improve responsive layout

refactor: simplify app initialization
```

---

# Coding Standards

* Use semantic HTML5
* Keep CSS modular
* Prefer vanilla JavaScript
* Write descriptive commit messages
* Keep documentation updated
* Test before committing

---

# Documentation First

Every major feature should include:

* Updated documentation
* Release notes
* Project status
* Architecture update (if applicable)

---

# Code Review Checklist

* Build succeeds
* Responsive layout verified
* JavaScript tested
* Documentation updated
* No console errors
* Clean Git history

---

Thank you for helping improve the iRAM Experience project.
